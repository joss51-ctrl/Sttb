\# Product: WebPage and CMS— Figma AI Brief

\#\# 1\) Goal  
\-Design a comprehensive, enterprise-grade web pages for a University that focuses on readability where user are comfortable to jump between pages.

\-Design a responsive, enterprise-grade web application for a \*\*Content Management System (CMS)\*\* used to manage website/app content, media, page structures, publishing workflows, and access control.

The design should prioritize:

\- Clear content hierarchy and easy content editing  
\- Fast navigation between content, media, pages, and settings  
\- Scalable management for multiple content types and sections  
\- Visibility of publishing status, approvals, and auditability  
\- Usable dashboards for content performance and operational overview  
\- Structured tables and forms for heavy back-office usage  
\- Role-based access and administration flows

\#\# 2\) Primary Users

\- \*\*Content Admin\*\*: Manages pages, content blocks, banners, articles, and publishing schedules.  
\- \*\*Editor\*\*: Creates and updates content drafts, reviews revisions, and submits for approval.  
\- \*\*Approver / Reviewer\*\*: Reviews submitted content and approves, rejects, or requests revisions.  
\- \*\*Super Admin\*\*: Manages users, permissions, system settings, and audit visibility.  
\- \*\*User\*\*: Viewer of the webpage

\#\# 4\) Design Style

\- Tone: academic, clean, modern, informative  
\- Layout: 12-column grid, sticky navbar, generous whitespace while maximizing screen utilization, topbar with search, notifications, and user profile  
\- Colors: neutral foundation with semantic accents for Success, Warning, Error, and Info states  
\- Components: summary cards, data tables, forms, tabs, breadcrumbs, drawers, modals, filters, chips/tags, pagination, empty states, upload zones  
\- Responsive: desktop-first, adaptive across common laptop and large monitor resolutions, tablet-friendly where practical  
\- Density: For CMS, optimized for admin usage, meaning layouts can be information-rich but should remain readable and structured.  
For WebPage, focus more on readability and utilize white spaces so that it doesn't feel cluttered

\#\# 5\) Accessibility Notes

\- All interactive elements must have visible focus states  
\- Ensure contrast-compliant text and status indicators  
\- Do not rely on color alone for statuses; pair with labels/icons  
\- Tables should support keyboard-friendly interaction patterns  
\- Forms should include clear labels, helper text, validation states, and error messaging  
\- Support switchable light and dark theme  
\- Use clear hierarchy for headings, section labels, and field groupings

\#\# 6\) Deliverables

\- Desktop frames for all main CMS pages  
\- Adaptable frames for all webpages  
\- Reusable component set with variants (status, size, state)  
\- Design tokens for spacing, radius, typography, elevation, and color roles  
\- Page templates for list pages, detail pages, form pages, and approval flows  
\- Responsive behavior guidance for major layouts and tables

\#\# 7\) Additional Instruction

\- Keep refactoring to maintain clean and scalable code structure  
\- Preserve finished elements and do not modify them unless explicitly requested  
\- Keep file sizes small and place helper functions and reusable components in separate files  
\- Date formats should be in the year \*\*2026\*\* for mock data and examples  
\- All generated code should use \*\*Next.js\*\* with \*\*Tailwind CSS\*\*  
\- Keep all UI elements aligned and non-overlapping unless explicitly instructed otherwise  
\- Only use absolute positioning when truly necessary; prefer responsive layouts using flexbox and grid  
\- Prioritize reusable admin patterns over one-off visual treatments  
\- Use realistic CMS sample content such as banners, landing pages, articles, FAQs, announcements, and media assets  
\- Include clear status patterns such as Draft, In Review, Approved, Published, Scheduled, and Rejected  
\- For tables, include practical columns such as title, content type, author, updated date, status, and actions  
\- For forms, include common CMS controls such as text input, rich text editor area, dropdowns, toggles, date pickers, image upload, and preview actions  
\- Avoid decorative clutter; emphasize clarity, workflow efficiency, and maintainability
